### Висновки

**Який формат найкраще підходить для якого типу зображень:**
Для фотографій можна використовувати формати із втратою якості (наприклад, MozJPEG, WebP (lossy), AVIF). Але бажано не опускати якість нижче за 75%, особливо у випадку MozJPEG (з'являються артефакти). Фотографії будуть гарно зжаті та можна буде отримати гарний результат із невеликою втратою якості.

Для скриншотів або зображень із текстом бажано використовувати lossless формати, такі як WebP (lossless) та PNG. В цих форматах зазвичай дуже важлива чіткість, але потрібно враховувати, що вага файлів значно більша за lossy варіанти.

В цьому порівнянні можна побачити, що найкращі результати отримував формат AVIF, маючи найменші розмірі файлів, при цьому не дуже сильно втрачаючи якість.

Стисненню краще всього піддавалась картинка скриншоту з форматом .png, фотографія та текстове зображення в форматаз .jpg отримували не дуже гарні результати, а іноді більше, ніж початковий варіант.

**Як зміна розміру впливає на оптимізацію:**
Зміно розміру може значно зменшити розміри файлу та не сильно знизити його якість. Але, це не завжди гарний варіант, оскільки при зменшенні деталі картинки знищуються, це критично особливо для тексту. Також потрібно не забувати про користувачів Retina дісплеїв, оскільки їх екрани побудовані так, що їм для чіткого зображення початкове зображення необхідно збільшити в 2 або 3 рази.

**Як правильно адаптувати зображення для Retina-дисплеїв:**
Необхідно збільшити зображення у 2 або 3 рази. Також можна перевести його в формат AVIF для зниження ваги з невеликою встратою якості, або використати інші формати за необхідності.
